# AI Coach

AI consultancy marketing site ŌĆö Astro + Tailwind, deployed to Cloudflare Workers via Static Assets.

## Local dev

```bash
npm install
npm run dev
```

Open http://localhost:4321.

## Build & preview

```bash
npm run build      # outputs to ./dist
npm run preview    # serves the built site locally
```

## Project structure

```
src/
  config/site.ts            ŌåÉ brand name, contact details, domain
  data/locations.ts         ŌåÉ counties + towns (drives /[county]/ and /[county]/[town]/)
  data/sectors.ts           ŌåÉ service lines (drives /services/[sector]/)
  content/blog/*.md         ŌåÉ blog posts (Astro Content Collections)
  components/               ŌåÉ Header, Footer, Breadcrumbs, RelatedLinks, CTA, Head
  layouts/BaseLayout.astro  ŌåÉ semantic shell + JSON-LD schema
  lib/seo.ts                ŌåÉ schema.org builders (Organization, Breadcrumb, Service, FAQ, Article)
  pages/                    ŌåÉ Astro file-based routes
public/                     ŌåÉ static assets (robots.txt, favicon.svg)
wrangler.jsonc              ŌåÉ Cloudflare Workers Static Assets config
astro.config.mjs            ŌåÉ Astro + Tailwind + sitemap
```

### Adding a new town

1. Open `src/data/locations.ts`.
2. Add a new `Town` object to the relevant county's `towns` array.
3. Run `npm run build` ŌĆö Astro generates the new page and updates the sitemap.
4. The town is auto-linked from its county hub and from sibling towns.

### Adding a new county

1. Add a new `County` object in `src/data/locations.ts`.
2. New routes generate automatically. Update the Footer/Locations index copy if the wording needs adjusting.

### Adding a blog post

Drop a Markdown file in `src/content/blog/` with this frontmatter:

```yaml
---
title: "..."
description: "..."
pubDate: 2026-05-17
tags: ["AI"]
---
```

## Deploying to Cloudflare Workers via GitHub

Cloudflare's current recommended deploy path for Astro sites is **Workers with Static Assets**.

### One-time setup

1. Push this repo to GitHub.
2. In the Cloudflare dashboard, go to **Workers & Pages ŌåÆ Create ŌåÆ Workers ŌåÆ Import a repository**.
3. Pick this repo. Cloudflare auto-detects Astro and sets:
   - **Build command:** `npm run build`
   - **Deploy command:** `npx wrangler deploy`
4. Hit **Create & Deploy**.

From then on, every push to `main` will rebuild and ship automatically.

### Manual deploy (optional)

```bash
npm run build
npx wrangler deploy
```

You will need `wrangler login` the first time.

### Custom domain

In the Workers dashboard for this project, go to **Settings ŌåÆ Domains & Routes ŌåÆ Add ŌåÆ Custom Domain** and add your domain. Cloudflare handles the DNS + SSL.

Then update `SITE_URL` in `src/config/site.ts` to your real domain and rebuild.

## SEO checklist

- Semantic HTML5 throughout (`<header>`, `<main>`, `<nav>`, `<article>`, `<address>`, `<time>`)
- One `<h1>` per page; clear `<h2>`/`<h3>` hierarchy
- JSON-LD on every page: `ProfessionalService`, `WebSite`, `BreadcrumbList`
- Service pages add `Service` + `FAQPage` schema
- Location pages add `Service` (with `areaServed`) + `FAQPage`
- Blog posts add `Article` schema
- Canonical URLs, OG tags, Twitter cards, robots meta on every page
- `/sitemap-index.xml` auto-generated by `@astrojs/sitemap`
- `/robots.txt` references the sitemap
- Inter-linking: county ŌåÆ towns, town ŌåÆ sibling towns + sectors, sector ŌåÆ counties + other sectors
